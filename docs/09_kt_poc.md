# 공통 다운로드 URL 기반 이모티콘 배포 방식 검토

## 1. 추진 배경

KT에서는 지자체가 보유한 자체 문자 발송 시스템(UMS)을 활용하여 시민에게 이모티콘을 배포하는 방안을 검토하고 있다.

기존 사용자별 개별 URL 발급 방식이 아닌, 하나의 공통 다운로드 URL을 생성하여 전체 대상자에게 동일한 URL을 발송하는 구조를 PoC 대상으로 검토하고자 한다.

---

# 2. 제안 구조

## 전체 프로세스

### ① 지자체

* 시민 대상 발송 명단(예: 500명) 보유
* 자체 UMS(문자 발송 시스템)를 통해 문자 발송

↓

### ② OGQ

* 이모티콘 다운로드용 공통 URL 1개 생성

예시

```
https://chattingplus.ogq.me/xxxx
```

↓

### ③ 지자체

* 생성된 동일한 URL을 문자에 포함
* 대상자 500명에게 일괄 발송

↓

### ④ 사용자

* 문자 내 URL 접속
* 회원 여부 확인

  * 기존 회원 : 로그인
  * 비회원 : 회원가입 진행
* 로그인(또는 회원가입) 완료 후 이모티콘 다운로드

---

# 3. 검토 대상

본 PoC에서는 사용자별 개별 다운로드 URL을 생성하는 방식이 아니라, 하나의 공통 다운로드 URL을 생성하여 모든 대상자가 동일한 URL을 사용하는 방식을 검토한다.

---

# 4. 검토 요청 사항

## 4.1 공통 다운로드 URL 지원 여부

* 현재 서비스 구조에서 하나의 공통 다운로드 URL 생성이 가능한지 검토가 필요하다.

## 4.2 기존 선물하기 구조 활용 가능 여부

* 현재 제공 중인 선물하기 URL 생성 구조를 활용하여 동일한 방식으로 구현 가능한지 검토가 필요하다.

## 4.3 회원가입 이후 다운로드 플로우

* 비회원이 회원가입을 완료한 이후 현재 서비스 플로우에서 자연스럽게 다운로드까지 이어질 수 있는지 검토가 필요하다.

## 4.4 다운로드 이력 및 사용 현황 추적

다음 항목에 대한 트래킹 가능 여부를 확인한다.

* 실제 다운로드 수
* 다운로드 완료 여부
* 사용자 이용 현황
* 기타 운영에 필요한 통계 정보

## 4.5 URL 공유에 따른 접근 가능 여부

공통 URL을 사용할 경우 다음 사항을 확인한다.

* URL을 전달받은 대상 외 사용자의 다운로드 가능 여부
* 대상자 외 접근이 가능한 구조인지 여부

## 4.6 보안 및 정책 검토

공통 URL 방식이 가능할 경우 다음 정책의 필요성을 검토한다.

* 대상자 제한 정책
* URL 공유 방지 정책
* 다운로드 횟수 제한
* 인증 방식 적용 여부
* 기타 운영 정책

---

# 5. 검토 의견 요청

KT에서는 지자체가 자체 UMS를 이용하여 동일한 다운로드 URL을 포함한 문자를 발송하는 방식을 전제로 PoC를 검토하고 있다.

이에 따라 현재 서비스 구조에서 다음 사항에 대한 검토 의견을 요청드린다.

* 공통 다운로드 URL 생성 가능 여부
* 기존 선물하기 구조 활용 가능 여부
* 회원가입 이후 다운로드 플로우 적합성
* 다운로드 및 사용 현황 트래킹 가능 여부
* URL 공유 시 보안 및 접근 제어 방안
* 구현 시 추가적으로 고려해야 할 기술적·운영적 사항
