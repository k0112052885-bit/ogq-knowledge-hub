# 결정 게이트

[후보 비교](02_candidates.html) 문서에서 검토한 3가지 방식 중 최종안을 결정합니다.

## 결정 사항

**채택안: B. SDK 임베드**

개발 난이도와 통신사 협조 부담의 균형, 그리고 사용자 경험 유지 측면에서
SDK 임베드 방식을 1차 채택안으로 결정합니다.

## 결정 근거

| 평가 기준 | 가중치 | A. 직접 API | B. SDK 임베드 | C. 딥링크 |

| 통신사 협조 부담 | 20% | 3 | 6 | 9 |

## 승인 현황

- [x] 개발팀 검토 완료
- [x] 보안팀 검토 완료
- [ ] 사업팀 최종 승인
- [ ] 통신사 파트너 통보

## 후속 조치

1. 사업팀 승인 완료 후 SDK 배포 일정을 확정합니다.
2. 통신사 파트너사에 [파트너 문의서](04_partner_inquiry.html)를 발송합니다.
3. 승인 완료 시 본 문서의 상태를 `locked`로 변경합니다.

## 리스크

```
- SDK 버전 파편화로 인한 QA 부담 증가 가능성
- 통신사 앱 정책 변경 시 SDK 재배포 필요
```

> 본 문서는 사업팀 최종 승인 전까지 `review` 상태를 유지합니다.
