# 파트너 문의서

[결정 게이트](03_decision_gate.html)에서 채택된 SDK 임베드 방식과 관련하여
통신사 파트너사에 전달할 문의 내용을 정리한 초안입니다.

## 문의 배경

OGQ는 귀사의 멤버십 서비스를 통해 OGQ 구독 혜택을 제공하는 연동을 검토하고 있으며,
SDK 임베드 방식을 1차 채택안으로 결정했습니다. 아래 항목에 대한 확인을 요청드립니다.

## 확인 요청 항목

| 번호 | 항목 | 상세 내용 |
| --- | --- | --- |
| 1 | SDK 배포 채널 | 귀사 앱 빌드에 포함 가능한 배포 방식 (Maven/CocoaPods 등) |
| 2 | 사용자 식별 방식 | 통신사 회원 식별에 사용할 토큰 규격 |
| 3 | 테스트 환경 | 스테이징 환경 접근 가능 여부 및 일정 |
| 4 | 개인정보 처리 | 사용자 동의 절차 및 데이터 전달 범위 |
| 5 | 장애 대응 | 연동 장애 시 담당 채널 및 SLA |

## 예상 일정

```
1주차: 문의서 발송 및 1차 회신
2주차: 기술 미팅 및 API 스펙 확정
3~4주차: SDK 연동 개발
5주차: 통합 테스트
6주차: 정식 오픈
```

## 확인 체크리스트 (파트너사 회신용)

- [ ] SDK 배포 채널 확인
- [ ] 사용자 식별 토큰 규격 확인
- [ ] 테스트 환경 접근 권한 제공
- [ ] 개인정보 처리 방침 검토
- [ ] 장애 대응 담당자 지정
